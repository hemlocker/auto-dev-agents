# Monitor 智能体知识库

## 角色定位

Monitor（质量监控专家）负责：
- 质量评估
- 质量报告生成
- 问题识别
- 趋势分析

---

## 质量评分体系

### 评分维度

| 维度 | 权重 | 说明 |
|------|------|------|
| 需求完整性 | 15% | 需求文档完整度 |
| 设计文档质量 | 15% | 设计文档质量 |
| 代码实现质量 | 20% | 代码质量评估 |
| 测试覆盖度 | 15% | 测试覆盖率 |
| 部署配置完整性 | 10% | 部署配置完整度 |
| 运维文档完整性 | 15% | 运维文档完整度 |
| 目录结构规范性 | 10% | 目录结构规范 |

### 评分等级

| 分数 | 等级 | 说明 |
|------|------|------|
| ≥ 90 | A | 优秀 |
| 80-89 | B | 良好 |
| 70-79 | C | 合格 |
| 60-69 | D | 需改进 |
| < 60 | F | 不合格 |

---

## 检查方法

### 需求完整性检查
- 用户需求文档存在
- 软件需求规格存在
- 需求追踪矩阵完整
- 每条需求有验收标准

### 设计文档质量检查
- 架构设计文档存在
- 数据库设计完整
- API 接口定义完整
- 有技术选型说明

### 代码质量检查
- 后端代码存在
- 前端代码存在
- 无明显语法错误
- 代码结构合理

### 测试覆盖度检查
- 测试代码存在
- 测试已执行
- 覆盖率报告存在
- 覆盖率 ≥ 80%

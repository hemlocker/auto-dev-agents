# 问题工单：测试阶段流程缺失

**发现时间**: 2026-03-21 20:05  
**优先级**: P1  
**类型**: 流程改进

---

## 问题描述

Testing 智能体在测试阶段只完成了部分工作：

| 应做 | 实际 |
|------|------|
| 编写测试用例 | ✅ 完成 |
| 执行测试 | ❌ 未执行 |
| 生成测试报告 | ❌ 未生成 |
| 问题反馈到 tickets | ❌ 未反馈 |

---

## 影响

1. 无法验证代码质量
2. 测试覆盖率未知
3. 潜在 bug 未被发现

---

## 解决方案

### 方案 1: 改进 Testing 智能体提示词

在 `prompts/testing/system.md` 中增加：

```markdown
## 执行要求

1. 生成测试代码
2. 执行测试命令 (npm test)
3. 解析测试结果
4. 生成测试报告到 output/tests/测试报告.md
5. 失败用例写入 input/tickets/
```

### 方案 2: 增加验证步骤

在 Testing 阶段后增加验证智能体，检查测试是否真正执行。

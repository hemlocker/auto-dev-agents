## 2. 问题背景

### 2.1 当前 PDCA 流程状态

```
PLAN → DO → CHECK → ACT → (断裂)
                      ↓
                  无自动触发下一轮
```

### 2.2 发现的问题

1. **阶段完成无验证**: 各阶段完成后无自动验证机制
2. **任务未自动分配**: 优化方案中的任务负责人均为"待分配"
3. **无跟进机制**: 实施跟踪表无更新
4. **阶段转换手动**: 需要人工触发下一阶段

### 任务 2: 创建阶段验证脚本

**描述**: 为每个阶段创建验证脚本

**产出文件**:
- `scripts/validate-plan.sh`
- `scripts/validate-do.sh`
- `scripts/validate-check.sh`
- `scripts/validate-act.sh`

**示例** (validate-act.sh):
```bash
#!/bin/bash
# 验证 ACT 阶段完成

PROJECT_DIR="$1"
ACT_DIR="$PROJECT_DIR/output/optimizer"

# 检查 optimizer 目录是否存在
if [ ! -d "$ACT_DIR" ]; then
    echo "❌ ACT 阶段未完成：缺少 optimizer 目录"
    exit 1
fi

# 检查是否有输出文件
FILE_COUNT=$(find "$ACT_DIR" -type f -name "*.md" | wc -l)
if [ $FILE_COUNT -eq 0 ]; then
    echo "❌ ACT 阶段未完成：optimizer 目录为空"
    exit 1
fi

# 检查是否生成了新工单
TICKET_COUNT=$(find "$PROJECT_DIR/output/tickets" -type f -name "OPT-*.md" | wc -l)
if [ $TICKET_COUNT -eq 0 ]; then
    echo "⚠️  警告：未生成下一轮 OPT 工单"
fi

echo "✅ ACT 阶段验证通过 ($FILE_COUNT 个文件)"
exit 0
```

**验收**:
- [ ] 4 个验证脚本已创建
- [ ] 脚本可正确检测阶段完成状态

**负责人**: 系统管理员

### 任务 5: 实现 PDCA 自动流转

**描述**: 阶段验证通过后自动触发下一阶段

**实现方式**:
1. 创建 PDCA 编排脚本 `scripts/pdca-orchestrator.sh`
2. 按顺序执行各阶段
3. 每阶段完成后自动验证
4. 验证通过后自动触发下一阶段

**流程**:
```bash
PLAN → 验证 → 通过 → DO → 验证 → 通过 → CHECK → 验证 → 通过 → ACT → 验证 → 通过 → (下一轮 PLAN)
```

**验收**:
- [ ] PDCA 编排脚本可执行

# 软件需求规格说明书 (SRS)

## 1. 引言

### 1.1 目的

本文档定义设备管理系统的软件需求规格，为系统设计、开发和测试提供依据。

### 1.2 范围

本系统为企业内部设备信息管理系统，支持设备的增删改查操作。

### 1.3 术语定义

| 术语 | 定义 |
|------|------|
| CRUD | Create(创建)、Read(读取)、Update(更新)、Delete(删除) |
| 设备状态 | 设备当前使用状态：在用、闲置、报废 |

## 2. 系统概述

### 2.1 系统功能架构

```
设备管理系统
├── 设备列表模块
│   ├── 列表展示
│   ├── 搜索功能
│   └── 筛选功能
├── 设备新增模块
├── 设备编辑模块
├── 设备删除模块
└── 设备详情模块
```

### 2.2 用户角色

| 角色 | 权限描述 |
|------|----------|
| 设备管理员 | 全部 CRUD 权限 |
| 部门主管 | 查看权限 |
| 普通员工 | 查看个人设备权限 |

## 3. 功能需求规格

### 3.1 设备列表展示 (FR-001)

**需求描述：** 系统应展示设备列表，支持搜索和筛选

**输入：** 无/搜索关键词/筛选条件

**处理：**
- 从数据库查询设备数据
- 根据搜索条件过滤数据
- 根据筛选条件过滤数据

**输出：** 设备列表数据

**验收标准：**
- 列表正确显示所有设备字段
- 搜索功能可按关键词匹配
- 筛选功能可按设备类型、状态等条件过滤

### 3.2 新增设备 (FR-002)

**需求描述：** 系统应支持新增设备记录

**输入：** 设备信息表单

**处理：**
- 验证必填字段
- 验证设备编号唯一性
- 保存设备数据

**输出：** 新增成功/失败提示

**验收标准：**
- 必填字段验证有效
- 设备编号重复时提示错误
- 新增成功后列表实时更新

### 3.3 编辑设备 (FR-003)

**需求描述：** 系统应支持修改设备信息

**输入：** 设备 ID、修改后的设备信息

**处理：**
- 根据设备 ID 查询原数据
- 验证修改数据
- 更新设备数据

**输出：** 修改成功/失败提示

**验收标准：**
- 可修改除设备编号外的所有字段
- 修改后数据正确保存
- 列表实时更新显示

### 3.4 删除设备 (FR-004)

**需求描述：** 系统应支持删除设备记录

**输入：** 设备 ID

**处理：**
- 确认删除操作
- 执行删除

**输出：** 删除成功/失败提示

**验收标准：**
- 删除前有确认提示
- 删除后数据从列表移除
- 支持批量删除（可选）

### 3.5 设备详情查看 (FR-005)

**需求描述：** 系统应展示设备完整信息

**输入：** 设备 ID

**处理：**
- 根据设备 ID 查询完整数据
- 格式化展示

**输出：** 设备详情页面

**验收标准：**
- 展示全部 10 个字段信息
- 信息展示清晰易读

## 4. 数据需求规格

### 4.1 设备数据表结构

| 字段名 | 数据类型 | 长度 | 必填 | 说明 |
|--------|----------|------|------|------|
| device_id | VARCHAR | 50 | 是 | 设备编号（主键） |
| device_name | VARCHAR | 100 | 是 | 设备名称 |
| device_type | VARCHAR | 50 | 是 | 设备类型 |
| specification | VARCHAR | 200 | 否 | 规格型号 |
| purchase_date | DATE | - | 是 | 购买日期 |
| department | VARCHAR | 100 | 是 | 使用部门 |
| user_name | VARCHAR | 50 | 是 | 使用人 |
| status | VARCHAR | 20 | 是 | 设备状态 |
| location | VARCHAR | 200 | 否 | 存放位置 |
| remark | TEXT | - | 否 | 备注 |

### 4.2 数据验证规则

| 字段 | 验证规则 |
|------|----------|
| 设备编号 | 必填，唯一，长度 1-50 字符 |
| 设备名称 | 必填，长度 1-100 字符 |
| 设备类型 | 必填，长度 1-50 字符 |
| 购买日期 | 必填，有效日期格式 |
| 使用部门 | 必填，长度 1-100 字符 |
| 使用人 | 必填，长度 1-50 字符 |
| 设备状态 | 必填，枚举值：在用/闲置/报废 |

## 5. 界面需求规格

### 5.1 界面设计原则

- 简洁易用，减少用户学习成本
- 响应式设计，适配 PC 和移动端
- 操作反馈及时明确

### 5.2 主要界面

| 界面名称 | 功能描述 |
|----------|----------|
| 设备列表页 | 展示设备列表、搜索、筛选、操作入口 |
| 设备新增页 | 设备信息录入表单 |
| 设备编辑页 | 设备信息修改表单 |
| 设备详情页 | 设备完整信息展示 |

## 6. 非功能需求

### 6.1 性能需求

- 列表加载时间 < 2 秒
- 操作响应时间 < 1 秒

### 6.2 可用性需求

- 系统可用性 > 99%
- 支持主流浏览器访问

### 6.3 安全需求

- 用户身份验证
- 操作权限控制
- 数据备份机制

## 7. 接口需求

### 7.1 内部接口

| 接口名称 | 方法 | 描述 |
|----------|------|------|
| /api/devices | GET | 获取设备列表 |
| /api/devices | POST | 新增设备 |
| /api/devices/{id} | GET | 获取设备详情 |
| /api/devices/{id} | PUT | 更新设备 |
| /api/devices/{id} | DELETE | 删除设备 |

## 8. 验收标准

- 所有功能需求实现并通过测试
- 数据验证机制有效
- 界面符合响应式设计要求
- 性能指标达标

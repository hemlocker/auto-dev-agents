## 2. 项目背景

企业需要一个简单的设备管理系统，用于统一管理企业内部设备信息，提高设备管理效率，实现设备信息的规范化、数字化管理。

## 3. 用户群体

- 设备管理员：负责设备信息的录入、维护和更新
- 部门主管：查看本部门设备使用情况
- 普通员工：查看个人名下设备信息

## 4. 业务目标

- 实现设备信息的集中化管理
- 提高设备查询和统计效率
- 明确设备使用责任
- 支持设备全生命周期管理

## 5. 功能需求

### 5.1 设备信息管理

| 需求编号 | 需求名称 | 需求描述 | 优先级 |
|----------|----------|----------|--------|
| UR-001 | 设备列表展示 | 用户可查看设备列表，支持搜索和筛选功能 | P0 |
| UR-002 | 新增设备 | 用户可录入新设备信息 | P0 |
| UR-003 | 编辑设备 | 用户可修改已有设备信息 | P0 |
| UR-004 | 删除设备 | 用户可删除设备记录 | P0 |
| UR-005 | 设备详情查看 | 用户可查看设备完整信息 | P0 |
| UR-006 | 使用人搜索 | 用户可通过使用人姓名搜索设备，支持模糊匹配 | P1 |

### 5.2 设备字段定义

| 字段序号 | 字段名称 | 字段说明 | 是否必填 |
|----------|----------|----------|----------|
| 1 | 设备编号 | 设备的唯一标识符 | 是 |
| 2 | 设备名称 | 设备的名称 | 是 |
| 3 | 设备类型 | 设备的分类类型 | 是 |
| 4 | 规格型号 | 设备的规格参数 | 否 |
| 5 | 购买日期 | 设备的采购时间 | 是 |
| 6 | 使用部门 | 设备所属部门 | 是 |
| 7 | 使用人 | 设备责任人 | 是 |
| 8 | 设备状态 | 在用/闲置/报废 | 是 |
| 9 | 存放位置 | 设备所在位置 | 否 |
| 10 | 备注 | 其他说明信息 | 否 |

## 6. 非功能需求

### 6.1 界面要求

- 界面简洁易用，符合用户操作习惯
- 采用响应式设计，支持多种设备访问

### 6.2 数据要求

- 支持数据验证，确保录入数据准确性
- 设备编号必须唯一

## 7. 验收标准

- 所有 P0 优先级功能正常可用
- 设备 CRUD 操作完整实现
- 搜索和筛选功能正常工作
- 数据验证机制有效
- 使用人搜索功能支持模糊匹配，搜索结果准确

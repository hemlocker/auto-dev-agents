# 设备管理系统 - 日志配置说明

## 日志架构

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│   Frontend  │────▶│    Nginx    │────▶│   Backend   │
│   (Vue 3)   │     │   (Proxy)   │     │  (Node.js)  │
└─────────────┘     └─────────────┘     └─────────────┘
       │                   │                   │
       ▼                   ▼                   ▼
  browser console    access/error.log    app.log + stdout
```

## Nginx 日志配置

### 当前配置位置
- 访问日志：`/var/log/nginx/access.log` (容器内)
- 错误日志：`/var/log/nginx/error.log` (容器内)

### 日志格式
```nginx
log_format main '$remote_addr - $remote_user [$time_local] "$request" '
                '$status $body_bytes_sent "$http_referer" '
                '"$http_user_agent" "$http_x_forwarded_for" '
                'rt=$request_time uct="$upstream_connect_time" '
                'uht="$upstream_header_time" urt="$upstream_response_time"';
```

### 日志轮转配置
创建 `/etc/logrotate.d/nginx`：
```
/var/log/nginx/*.log {
    daily
    missingok
    rotate 14
    compress
    delaycompress
    notifempty
    create 0640 www-data adm
    sharedscripts
    postrotate
        [ -f /var/run/nginx.pid ] && kill -USR1 `cat /var/run/nginx.pid`
    endscript
}
```

## Node.js 后端日志配置

### 日志级别
- `error`: 错误和异常
- `warn`: 警告信息
- `info`: 一般信息
- `debug`: 调试信息 (生产环境禁用)

### 日志输出
- 标准输出：stdout (Docker 日志收集)
- 文件输出：`/app/data/logs/app.log`

### 推荐日志库配置
在 `backend/src/app.js` 中添加：
```javascript
const winston = require('winston');
const path = require('path');

const logger = winston.createLogger({
  level: process.env.LOG_LEVEL || 'info',
  format: winston.format.combine(
    winston.format.timestamp({ format: 'YYYY-MM-DD HH:mm:ss' }),
    winston.format.errors({ stack: true }),
    winston.format.splat(),
    winston.format.json()
  ),
  defaultMeta: { service: 'device-management' },
  transports: [
    // 控制台输出
    new winston.transports.Console({
      format: winston.format.simple()
    }),
    // 文件输出
    new winston.transports.File({
      filename: path.join(process.env.DATA_PATH || '/app/data', 'logs', 'error.log'),
      level: 'error',
      maxsize: 10485760, // 10MB
      maxFiles: 5
    }),
    new winston.transports.File({
      filename: path.join(process.env.DATA_PATH || '/app/data', 'logs', 'combined.log'),
      maxsize: 10485760,
      maxFiles: 5
    })
  ]
});
```

## Docker 日志配置

### docker-compose.yml 日志配置
更新 `docker-compose.yml` 添加日志驱动：
```yaml
services:
  frontend:
    # ... 其他配置
    logging:
      driver: "json-file"
      options:
        max-size: "10m"
        max-file: "3"

  backend:
    # ... 其他配置
    logging:
      driver: "json-file"
      options:
        max-size: "10m"
        max-file: "5"
```

### 查看 Docker 日志
```bash
# 查看所有服务日志
docker-compose logs

# 实时跟踪日志
docker-compose logs -f

# 查看最近 100 行
docker-compose logs --tail=100

# 查看特定时间范围
docker-compose logs --since="2026-03-21T10:00:00" --until="2026-03-21T12:00:00"

# 导出日志到文件
docker-compose logs > logs_$(date +%Y%m%d_%H%M%S).txt
```

## 日志收集方案

### 方案一：ELK Stack (推荐)
```yaml
# docker-compose.logging.yml
version: '3.8'
services:
  elasticsearch:
    image: docker.elastic.co/elasticsearch/elasticsearch:8.11.0
    environment:
      - discovery.type=single-node
      - xpack.security.enabled=false
    volumes:
      - es-data:/usr/share/elasticsearch/data
    ports:
      - "9200:9200"

  logstash:
    image: docker.elastic.co/logstash/logstash:8.11.0
    volumes:
      - ./logstash/pipeline:/usr/share/logstash/pipeline
    depends_on:
      - elasticsearch

  kibana:
    image: docker.elastic.co/kibana/kibana:8.11.0
    ports:
      - "5601:5601"
    depends_on:
      - elasticsearch
```

### 方案二：Loki + Promtail (轻量级)
```yaml
# docker-compose.loki.yml
version: '3.8'
services:
  loki:
    image: grafana/loki:2.9.0
    ports:
      - "3100:3100"
    command: -config.file=/etc/loki/local-config.yaml

  promtail:
    image: grafana/promtail:2.9.0
    volumes:
      - /var/log:/var/log
      - /var/lib/docker/containers:/var/lib/docker/containers:ro
    command: -config.file=/etc/promtail/config.yml
```

## 日志分析命令

### 错误统计
```bash
# 统计错误数量
docker-compose logs | grep -i error | wc -l

# 查看最近 10 个错误
docker-compose logs | grep -i error | tail -10

# 按状态码统计
docker-compose logs frontend | grep -oP 'HTTP/1.1" \K\d{3}' | sort | uniq -c
```

### 访问分析
```bash
# 统计 Top 10 IP
docker-compose logs frontend | grep -oP '^\S+' | sort | uniq -c | sort -rn | head -10

# 统计 Top 10 请求路径
docker-compose logs frontend | grep -oP 'GET \S+|POST \S+' | sort | uniq -c | sort -rn | head -10

# 统计每小时请求量
docker-compose logs --since="1h" | grep -c "GET\|POST"
```

## 告警规则

### 基于日志的告警
1. **错误率告警**: 5 分钟内错误日志 > 10 条
2. **响应时间告警**: 平均响应时间 > 2 秒
3. **服务不可用告警**: 健康检查失败连续 3 次
4. **磁盘空间告警**: 日志目录占用 > 80%

### 告警配置示例 (Prometheus Alertmanager)
```yaml
groups:
  - name: logs
    rules:
      - alert: HighErrorRate
        expr: rate(container_logs_total{level="error"}[5m]) > 0.5
        for: 5m
        labels:
          severity: warning
        annotations:
          summary: "高错误率"
          description: "应用错误率超过阈值"
```

## 日志保留策略

| 日志类型 | 保留期限 | 存储位置 |
|---------|---------|---------|
| 访问日志 | 30 天 | 本地 + 归档 |
| 错误日志 | 90 天 | 本地 + 归档 |
| 审计日志 | 1 年 | 归档存储 |
| 调试日志 | 7 天 | 本地 |

## 安全注意事项

1. **敏感信息脱敏**: 日志中不得包含密码、token 等敏感信息
2. **访问控制**: 日志文件权限设置为 640
3. **传输加密**: 日志收集使用 TLS 加密
4. **审计追踪**: 记录所有日志访问操作

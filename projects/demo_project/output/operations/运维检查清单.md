## 检查命令

### 服务状态
```bash
# 查看所有容器状态
docker-compose ps

# 查看容器详细信息
docker inspect demo_project_backend_1
docker inspect demo_project_frontend_1
```

### 日志查看
```bash
# 实时查看日志
docker-compose logs -f

# 查看最近 100 行日志
docker-compose logs --tail=100

# 查看后端日志
docker-compose logs backend

# 查看前端日志
docker-compose logs frontend
```

### 资源监控
```bash
# 查看容器资源使用
docker stats

# 查看磁盘使用
docker system df

# 查看系统资源
free -h
df -h
```

### 健康检查
```bash
# 检查前端服务
curl -I http://localhost

# 检查后端 API
curl http://localhost/api/health

# 检查数据库连接
docker exec demo_project_backend_1 npm run test:db
```

## 应急预案

### 服务不可用
1. 检查容器状态：`docker-compose ps`
2. 查看错误日志：`docker-compose logs --tail=200`
3. 重启服务：`docker-compose restart`
4. 如无法恢复，执行回滚流程

### 数据库异常
1. 停止写入操作
2. 检查数据库连接：`docker exec demo_project_backend_1 ls -la /app/data`
3. 从备份恢复数据
4. 验证数据完整性

### 性能下降
1. 检查资源使用：`docker stats`
2. 分析慢查询日志
3. 清理缓存和临时文件
4. 考虑水平扩容

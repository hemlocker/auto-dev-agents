# 新功能实现说明

## 概述

本次实现为设备管理系统添加了三个核心新功能：
1. **Excel 导入导出功能** - 支持批量导入和导出数据
2. **设备标签打印功能** - 支持单台和批量打印设备标签（含二维码）
3. **统计报表功能** - 支持多维度统计分析和图表展示

---

## 快速开始

### 1. 安装依赖

```bash
# 安装后端依赖
cd backend
npm install

# 安装前端依赖
cd ../frontend
npm install
```

### 2. 启动服务

```bash
# 启动后端（端口 3000）
cd backend
npm start

# 启动前端（端口 5173）
cd ../frontend
npm run dev
```

## 功能使用说明

### Excel 导入导出

#### 访问路径
从设备列表页点击「导入导出」按钮，或访问 `/device/import-export`

#### 导入流程
1. **下载模板** - 点击"下载模板"按钮获取 Excel 模板
2. **填写数据** - 按模板要求填写设备信息
3. **上传文件** - 拖拽或选择填写好的 Excel 文件
4. **查看结果** - 系统显示导入成功/失败统计

#### 导出流程
1. 选择导出范围（全部设备或按筛选条件）
2. 设置筛选条件（类型、状态、部门、关键词）
3. 点击"确认导出"下载 Excel 文件

#### 数据验证规则
- 设备编号：必填，唯一，1-50 字符，仅限字母/数字/下划线/中划线
- 设备名称：必填，1-100 字符
- 设备类型：必填，使用预定义类型
- 购买日期：必填，YYYY-MM-DD 格式，不能晚于今日
- 使用部门：必填，1-100 字符
- 使用人：必填，1-50 字符
- 设备状态：必填，仅限"在用"/"闲置"/"报废"
- 存放位置：可选，最多 200 字符
- 备注：可选，最多 500 字符

---

### 设备标签打印

#### 单台打印
1. 在设备列表页点击某设备的"打印"按钮
2. 查看标签预览（含二维码）
3. 设置打印参数（纸张大小、每页数量）
4. 点击"打印"调用浏览器打印或下载 PDF

#### 批量打印
1. 在设备列表页选择多个设备（待实现复选框）
2. 点击"批量打印"按钮
3. 系统生成包含所有设备标签的 PDF 文件
4. 下载并打印 PDF

#### 标签内容
- 设备编号（醒目显示）
- 设备名称
- 设备类型
- 规格型号
- 使用部门
- 使用人
- 购买日期
- 设备状态
- 二维码（扫码查看设备详情）

#### 标签规格
- 标准尺寸：90mm × 60mm
- A4 纸布局：每页 8 个（2 列 × 4 行）

---

### 统计报表

#### 访问路径
从设备列表页点击「统计报表」按钮，或访问 `/reports`

#### 统计维度

**1. 按设备状态统计**
- 展示在用、闲置、报废设备数量和占比
- 饼图可视化分布
- 摘要卡片显示详细统计

**2. 按使用部门统计**
- 展示各部门设备数量
- 柱状图对比显示
- 支持按数量排序

**3. 按设备类型统计**
- 展示各类设备占比
- 饼图可视化分布
- 识别主要设备类型

**4. 按购买时间统计**
- 展示设备采购趋势
- 折线图显示变化
- 支持月度/年度自动切换

#### 时间筛选
- 支持自定义时间范围
- 长期数据自动按年统计（>2 年）
- 短期数据按月统计

#### 图表交互
- 鼠标悬停显示详细数据
- 图例点击切换显示
- 窗口大小自适应

## API 接口文档

### 导入导出接口

#### 下载导入模板
```
GET /api/devices/import-template
```
**响应**: Excel 文件下载

#### 批量导入设备
```
POST /api/devices/import
Content-Type: multipart/form-data

参数：file (Excel 文件)
```
**响应示例**:
```json
{
  "code": 200,
  "message": "导入完成",
  "data": {
    "total": 50,
    "success": 48,
    "failed": 2,
    "errors": [
      {
        "row": 5,
        "device_id": "DEV005",
        "reason": "设备编号已存在"
      }
    ]
  }
}
```

#### 导出设备数据
```
GET /api/devices/export
参数：
  - keyword: 搜索关键词
  - device_type: 设备类型
  - status: 设备状态
  - department: 使用部门
```
**响应**: Excel 文件下载

---

### 打印接口

#### 获取设备标签数据
```
GET /api/devices/:id/label
```
**响应示例**:
```json
{
  "code": 200,
  "message": "success",
  "data": {
    "device_id": "DEV001",
    "device_name": "联想 ThinkPad X1",
    "device_type": "电脑",
    "specification": "i7/16G/512G",
    "purchase_date": "2025-01-15",
    "department": "技术部",
    "user_name": "张三",
    "status": "在用",
    "qr_code": "data:image/png;base64,..."
  }
}
```

#### 批量打印设备标签
```
POST /api/devices/print-batch
Body:
{
  "device_ids": ["DEV001", "DEV002", "DEV003"]
}
```
**响应**: PDF 文件下载

---

### 统计报表接口

#### 获取统计数据
```
GET /api/reports/statistics
参数:
  - dimension: 统计维度（必填）
    - status: 按状态
    - department: 按部门
    - type: 按类型
    - time: 按时间
  - start_date: 开始日期（可选）
  - end_date: 结束日期（可选）
```

**响应示例（按状态）**:
```json
{
  "code": 200,
  "message": "success",
  "data": {
    "dimension": "status",
    "summary": {
      "total": 150,
      "in_use": 120,
      "idle": 25,
      "scrapped": 5
    },
    "details": [
      {
        "status": "在用",
        "count": 120,
        "percentage": 80.0
      },
      {
        "status": "闲置",
        "count": 25,
        "percentage": 16.7
      },
      {
        "status": "报废",
        "count": 5,
        "percentage": 3.3
      }
    ]
  }
}
```

## 注意事项

### 导入功能
- 仅支持 `.xlsx` 格式文件
- 文件大小限制：10MB
- 设备编号必须唯一，重复会导致导入失败
- 日期格式必须为 `YYYY-MM-DD`
- 建议先下载模板，按格式填写后再上传

### 打印功能
- 单台打印使用浏览器打印 API
- 批量打印生成 PDF 文件下载
- 二维码包含设备详情 URL，需要后端服务运行
- 建议使用 A4 纸打印，效果最佳

### 报表功能
- 首次加载图表可能需要 1-2 秒
- 大量数据时建议使用时间范围筛选
- 图表支持响应式，可调整窗口大小
- 导出功能预留接口，后续可实现

## 故障排查

### 导入失败
1. 检查 Excel 文件格式是否正确
2. 确认必填字段无空值
3. 验证日期格式是否为 YYYY-MM-DD
4. 检查设备编号是否重复
5. 查看错误报告中的具体原因

### 打印失败
1. 检查设备 ID 是否正确
2. 确认后端服务正常运行
3. 批量打印时检查设备 ID 列表格式
4. 浏览器拦截弹窗时允许下载

### 报表加载失败
1. 检查统计维度参数是否正确
2. 确认时间范围格式正确
3. 查看浏览器控制台错误信息
4. 验证后端 API 是否正常响应

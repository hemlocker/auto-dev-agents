## 2. 系统架构设计

### 2.1 整体架构图

```
┌─────────────────────────────────────────────────────────────┐
│                        前端层 (Frontend)                     │
│  ┌─────────────┐  ┌─────────────┐  ┌─────────────┐         │
│  │  设备列表页  │  │  设备新增页  │  │  设备详情页  │         │
│  └─────────────┘  └─────────────┘  └─────────────┘         │
│                    (Vue 3 + Element Plus)                    │
└─────────────────────────────────────────────────────────────┘
                              │
                              │ HTTP/REST API
                              ▼
┌─────────────────────────────────────────────────────────────┐
│                        后端层 (Backend)                      │
│  ┌─────────────────────────────────────────────────────┐    │
│  │              API Gateway / Controller               │    │
│  │  ┌─────────┐ ┌─────────┐ ┌─────────┐ ┌─────────┐   │    │
│  │  │ GET     │ │ POST    │ │ PUT     │ │ DELETE  │   │    │
│  │  │ /devices│ │ /devices│ │ /devices│ │ /devices│   │    │
│  │  └─────────┘ └─────────┘ └─────────┘ └─────────┘   │    │
│  └─────────────────────────────────────────────────────┘    │
│                    (Node.js + Express)                       │
└─────────────────────────────────────────────────────────────┘
                              │
                              │ SQL
                              ▼
┌─────────────────────────────────────────────────────────────┐
│                        数据层 (Database)                     │
│  ┌─────────────────────────────────────────────────────┐    │
│  │                  devices 表                          │    │
│  │  (device_id, device_name, device_type, ...)         │    │
│  └─────────────────────────────────────────────────────┘    │
│                      (SQLite / MySQL)                        │
└─────────────────────────────────────────────────────────────┘
```

### 2.2 技术栈选择

| 层级 | 技术选型 | 说明 |
|------|----------|------|
| **前端** | Vue 3 + Vite | 轻量、快速、易上手 |
| **UI 组件库** | Element Plus | 成熟的 Vue3 组件库，支持响应式 |
| **后端** | Node.js + Express | 简单高效，适合 CRUD 场景 |
| **数据库** | SQLite (开发) / MySQL (生产) | 开发便捷，生产可扩展 |
| **ORM** | Sequelize | 支持多数据库，简化数据操作 |
| **API 规范** | RESTful | 标准、简洁、易理解 |

### 2.3 架构特点

- **前后端分离**：便于独立开发和部署
- **RESTful API**：标准接口设计，易于理解和维护
- **简洁实用**：避免过度设计，满足核心需求
- **可扩展性**：预留扩展空间，支持后续功能迭代

## 3. 数据库设计

### 3.1 设备表 (devices)

| 序号 | 字段名 | 数据类型 | 长度 | 必填 | 默认值 | 说明 |
|------|--------|----------|------|------|--------|------|
| 1 | device_id | VARCHAR | 50 | 是 | - | 设备编号（主键） |
| 2 | device_name | VARCHAR | 100 | 是 | - | 设备名称 |
| 3 | device_type | VARCHAR | 50 | 是 | - | 设备类型 |
| 4 | specification | VARCHAR | 200 | 否 | NULL | 规格型号 |
| 5 | purchase_date | DATE | - | 是 | - | 购买日期 |
| 6 | department | VARCHAR | 100 | 是 | - | 使用部门 |
| 7 | user_name | VARCHAR | 50 | 是 | - | 使用人 |
| 8 | status | VARCHAR | 20 | 是 | '闲置' | 设备状态（在用/闲置/报废） |
| 9 | location | VARCHAR | 200 | 否 | NULL | 存放位置 |
| 10 | remark | TEXT | - | 否 | NULL | 备注 |

### 3.2 SQL 建表语句

```sql
CREATE TABLE devices (
    device_id VARCHAR(50) PRIMARY KEY,
    device_name VARCHAR(100) NOT NULL,
    device_type VARCHAR(50) NOT NULL,
    specification VARCHAR(200),
    purchase_date DATE NOT NULL,
    department VARCHAR(100) NOT NULL,
    user_name VARCHAR(50) NOT NULL,
    status VARCHAR(20) NOT NULL DEFAULT '闲置',
    location VARCHAR(200),
    remark TEXT,
    
    -- 索引
    INDEX idx_device_type (device_type),
    INDEX idx_status (status),
    INDEX idx_department (department),
    INDEX idx_user_name (user_name)
);
```

### 3.3 数据字典

#### 设备状态 (status)

| 值 | 说明 |
|----|------|
| 在用 | 设备正在使用中 |
| 闲置 | 设备未被使用 |
| 报废 | 设备已报废 |

#### 设备类型 (device_type)

建议预定义类型（可扩展）：
- 电脑
- 打印机
- 投影仪
- 办公桌椅
- 网络设备
- 其他

## 4. API 接口设计

### 4.1 接口规范

- **基础路径**: `/api`
- **数据格式**: JSON
- **字符编码**: UTF-8
- **响应格式**: 统一响应结构

### 4.2 统一响应结构

```json
{
  "code": 200,
  "message": "success",
  "data": {}
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| code | Number | 状态码（200=成功，其他=失败） |
| message | String | 响应消息 |
| data | Object/Array | 响应数据 |

### 4.3 接口列表

#### 4.3.1 获取设备列表

| 项目 | 内容 |
|------|------|
| 接口路径 | `/api/devices` |
| 请求方法 | GET |
| 描述 | 获取设备列表，支持搜索和筛选 |

**请求参数:**

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| keyword | String | 否 | 搜索关键词（匹配设备名称/编号） |
| device_type | String | 否 | 设备类型筛选 |
| status | String | 否 | 设备状态筛选 |
| department | String | 否 | 使用部门筛选 |
| page | Number | 否 | 页码（默认 1） |
| pageSize | Number | 否 | 每页数量（默认 10） |

**响应示例:**

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "list": [
      {
        "device_id": "DEV001",
        "device_name": "联想 ThinkPad X1",
        "device_type": "电脑",
        "specification": "i7/16G/512G",
        "purchase_date": "2025-01-15",
        "department": "技术部",
        "user_name": "张三",
        "status": "在用",
        "location": "A 栋 301",
        "remark": ""
      }
    ],
    "total": 1,
    "page": 1,
    "pageSize": 10
  }
}
```

#### 4.3.2 新增设备

| 项目 | 内容 |
|------|------|
| 接口路径 | `/api/devices` |
| 请求方法 | POST |
| 描述 | 新增设备记录 |

**请求体:**

```json
{
  "device_id": "DEV002",
  "device_name": "惠普打印机",
  "device_type": "打印机",
  "specification": "LaserJet Pro",
  "purchase_date": "2025-03-20",
  "department": "行政部",
  "user_name": "李四",
  "status": "闲置",
  "location": "B 栋 102",
  "remark": "新采购"
}
```

**响应示例:**

```json
{
  "code": 200,
  "message": "新增成功",
  "data": {
    "device_id": "DEV002"
  }
}
```

**错误响应:**

```json
{
  "code": 400,
  "message": "设备编号已存在",
  "data": null
}
```

## 5. 前端页面设计

### 5.1 页面结构

```
设备管理系统
├── 设备列表页 (/)
│   ├── 搜索栏
│   ├── 筛选条件
│   ├── 设备表格
│   └── 分页器

    └── 设备信息展示
```

### 5.2 设备列表页

**功能模块:**

1. **搜索栏**
   - 关键词输入框（搜索设备名称/编号）
   - 搜索按钮
   - 重置按钮

2. **筛选条件**
   - 设备类型下拉框
   - 设备状态下拉框
   - 使用部门下拉框

3. **操作按钮**
   - 新增设备按钮
   - 批量删除按钮（可选）

4. **设备表格**
   - 显示字段：设备编号、设备名称、设备类型、使用部门、使用人、设备状态、操作
   - 操作列：查看、编辑、删除

5. **分页器**
   - 页码显示
   - 每页数量选择

**界面布局:**

### 2.2 技术栈选择

| 层级 | 技术选型 | 说明 |
|------|----------|------|
| **前端** | Vue 3 + Vite | 轻量、快速、易上手 |
| **UI 组件库** | Element Plus | 成熟的 Vue3 组件库，支持响应式 |
| **后端** | Node.js + Express | 简单高效，适合 CRUD 场景 |
| **数据库** | SQLite (开发) / MySQL (生产) | 开发便捷，生产可扩展 |
| **ORM** | Sequelize | 支持多数据库，简化数据操作 |
| **API 规范** | RESTful | 标准、简洁、易理解 |
| **Excel 处理** | SheetJS (xlsx) | 前端导出/后端导入导出 |
| **PDF 生成** | pdfmake / Puppeteer | 标签打印和报表导出 |
| **打印支持** | 浏览器打印 API | 调用系统打印机 |

### 2.3 架构特点

- **前后端分离**：便于独立开发和部署
- **RESTful API**：标准接口设计，易于理解和维护
- **简洁实用**：避免过度设计，满足核心需求
- **可扩展性**：预留扩展空间，支持后续功能迭代
- **模块化设计**：导入导出、打印、报表独立模块，便于维护

## 3. 数据库设计

### 3.1 设备表 (devices)

| 序号 | 字段名 | 数据类型 | 长度 | 必填 | 默认值 | 说明 |
|------|--------|----------|------|------|--------|------|
| 1 | device_id | VARCHAR | 50 | 是 | - | 设备编号（主键） |
| 2 | device_name | VARCHAR | 100 | 是 | - | 设备名称 |
| 3 | device_type | VARCHAR | 50 | 是 | - | 设备类型 |
| 4 | specification | VARCHAR | 200 | 否 | NULL | 规格型号 |
| 5 | purchase_date | DATE | - | 是 | - | 购买日期 |
| 6 | department | VARCHAR | 100 | 是 | - | 使用部门 |
| 7 | user_name | VARCHAR | 50 | 是 | - | 使用人 |
| 8 | status | VARCHAR | 20 | 是 | '闲置' | 设备状态（在用/闲置/报废） |
| 9 | location | VARCHAR | 200 | 否 | NULL | 存放位置 |
| 10 | remark | TEXT | - | 否 | NULL | 备注 |
| 11 | created_at | DATETIME | - | 是 | NOW() | 创建时间 |
| 12 | updated_at | DATETIME | - | 是 | NOW() | 更新时间 |

### 3.2 SQL 建表语句

```sql
CREATE TABLE devices (
    device_id VARCHAR(50) PRIMARY KEY,
    device_name VARCHAR(100) NOT NULL,
    device_type VARCHAR(50) NOT NULL,
    specification VARCHAR(200),
    purchase_date DATE NOT NULL,
    department VARCHAR(100) NOT NULL,
    user_name VARCHAR(50) NOT NULL,
    status VARCHAR(20) NOT NULL DEFAULT '闲置',
    location VARCHAR(200),
    remark TEXT,
    created_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    
    -- 索引
    INDEX idx_device_type (device_type),
    INDEX idx_status (status),
    INDEX idx_department (department),
    INDEX idx_user_name (user_name),
    INDEX idx_created_at (created_at)
);
```

### 3.3 数据字典

#### 设备状态 (status)

| 值 | 说明 |
|----|------|
| 在用 | 设备正在使用中 |
| 闲置 | 设备未被使用 |
| 报废 | 设备已报废 |

#### 设备类型 (device_type)

建议预定义类型（可扩展）：
- 电脑
- 打印机
- 投影仪
- 办公桌椅
- 网络设备
- 其他

## 4. API 接口设计

### 4.1 接口规范

- **基础路径**: `/api`
- **数据格式**: JSON
- **字符编码**: UTF-8
- **响应格式**: 统一响应结构

### 4.2 统一响应结构

```json
{
  "code": 200,
  "message": "success",
  "data": {}
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| code | Number | 状态码（200=成功，其他=失败） |
| message | String | 响应消息 |
| data | Object/Array | 响应数据 |

### 4.3 基础 CRUD 接口

#### 4.3.1 获取设备列表

| 项目 | 内容 |
|------|------|
| 接口路径 | `/api/devices` |
| 请求方法 | GET |
| 描述 | 获取设备列表，支持搜索和筛选 |

**请求参数:**

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| keyword | String | 否 | 搜索关键词（匹配设备名称/编号/使用人） |

| status | String | 否 | 设备状态筛选 |
| department | String | 否 | 使用部门筛选 |
| page | Number | 否 | 页码（默认 1） |
| pageSize | Number | 否 | 每页数量（默认 10） |

**响应示例:**

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "list": [
      {
        "device_id": "DEV001",
        "device_name": "联想 ThinkPad X1",
```

#### 4.4.2 批量导入设备

| 项目 | 内容 |
|------|------|
| 接口路径 | `/api/devices/import` |
| 请求方法 | POST |
| 描述 | 批量导入设备数据 |

**请求体:**
- Content-Type: `multipart/form-data`
- 参数：`file` (Excel 文件)

**处理流程:**
1. 解析上传的 Excel 文件
2. 验证数据格式和必填字段
3. 检查设备编号唯一性
4. 批量插入数据库
5. 返回导入结果报告

**响应示例:**

```json
{
  "code": 200,
  "message": "导入完成",
  "data": {
    "total": 50,
    "success": 48,
    "failed": 2,
    "errors": [
      {
        "row": 5,
        "device_id": "DEV005",
        "reason": "设备编号已存在"
      },
      {
        "row": 12,
        "device_id": "",
        "reason": "设备编号不能为空"
      }
    ]
  }
}
```

| 描述 | 导出统计报表为 Excel 或 PDF 文件 |

**请求参数:**

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| dimension | String | 是 | 统计维度：status/department/type/time |
| format | String | 是 | 导出格式：excel/pdf |
| start_date | String | 否 | 开始日期 |
| end_date | String | 否 | 结束日期 |

**响应:**
- Excel: Content-Type: `application/vnd.openxmlformats-officedocument.spreadsheetml.sheet`
- PDF: Content-Type: `application/pdf`
- Content-Disposition: `attachment; filename="统计报表_维度_日期.xlsx/pdf"`

|--------|------|

| 404 | 资源不存在 |

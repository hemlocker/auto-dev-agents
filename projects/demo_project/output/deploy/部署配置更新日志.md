## 配置文件更新详情

### 1. Dockerfile.backend 更新

**文件路径**: `/deploy/Dockerfile.backend`

**更新内容**:
```dockerfile
# 新增系统依赖安装（用于 pdfkit 和 qrcode）
RUN apk add --no-cache \
    fontconfig \
    freetype \
    cairo \
    pango \
    jpeg \
    giflib
```

**更新原因**:
- `pdfkit` 需要 Cairo、Pango、Freetype 等图形库支持
- `qrcode` 需要图像库支持
- 确保在 Alpine Linux 环境中正常生成 PDF 和二维码

**完整文件**:
```dockerfile
FROM node:18-alpine

WORKDIR /app

# 安装系统依赖 (用于 pdfkit 和 qrcode)
RUN apk add --no-cache \
    fontconfig \
    freetype \
    cairo \
    pango \
    jpeg \
    giflib

# 复制后端代码
COPY backend/package*.json ./
RUN npm install --production

COPY backend/ ./

# 暴露端口
EXPOSE 3000

# 启动命令
CMD ["npm", "start"]
```

---

### 2. docker-compose.yml 更新

**文件路径**: `/deploy/docker-compose.yml`

**更新内容**:
- 为 frontend 和 backend 服务添加 `restart: unless-stopped` 策略
- 确保容器在系统重启后自动恢复

**完整文件**:
```yaml
version: '3.8'

services:
  # 前端服务
  frontend:
    build:
      context: ../src
      dockerfile: ../../deploy/Dockerfile.frontend
    ports:
      - "80:80"
    depends_on:
      - backend
    networks:
      - app-network
    restart: unless-stopped

  # 后端服务
  backend:
    build:
      context: ../src
      dockerfile: ../../deploy/Dockerfile.backend
    expose:
      - "3000"
    volumes:
      - db-data:/app/data
    environment:
      - NODE_ENV=production
      - PORT=3000
    networks:
      - app-network
    restart: unless-stopped

networks:
  app-network:
    driver: bridge

volumes:
  db-data:
```

## 故障排查

### 问题 1: PDF 生成失败

**症状**: 打印标签时返回错误

**解决方案**:
```bash
# 检查系统依赖是否安装
docker-compose exec backend apk info | grep -E "cairo|pango"

# 重新构建镜像
docker-compose build --no-cache backend
docker-compose up -d backend
```

### 问题 2: 二维码无法生成

**症状**: 标签预览无二维码

**解决方案**:
```bash
# 检查 qrcode 依赖
docker-compose exec backend npm list qrcode

# 查看后端日志
docker-compose logs backend | grep -i "qrcode\|error"
```

### 问题 3: Excel 导入失败

**症状**: 上传 Excel 文件后报错

**解决方案**:
```bash
# 检查 xlsx 依赖
docker-compose exec backend npm list xlsx

# 检查文件上传限制
docker-compose logs backend | grep -i "multer\|upload"
```

### 问题 4: 图表无法加载

**症状**: 统计报表页面图表不显示

**解决方案**:
```bash
# 检查前端构建
docker-compose logs frontend | grep -i "error\|echarts"

# 重新构建前端
docker-compose build --no-cache frontend
docker-compose up -d frontend
```

## 性能优化建议

### 1. 镜像大小优化

当前后端镜像大小约 ~500MB（包含图形库）

**优化方案**:
- 使用多阶段构建
- 精简系统依赖
- 使用更小的基础镜像（如 distroless）

### 2. 启动时间优化

当前启动时间约 30-60 秒

**优化方案**:
- 使用预构建镜像
- 启用 Docker 层缓存
- 优化 npm install（使用 ci 代替 install）

### 3. 生产环境建议

- 使用反向代理（Nginx/Traefik）
- 配置 HTTPS
- 启用日志轮转
- 设置资源限制（CPU/内存）
- 配置健康检查
